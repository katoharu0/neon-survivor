このファイルは **ゲームの現在の仕様（数値・仕組み）と設計方針** をまとめたものです。

## 1. 概要

- **ジャンル**: ヴァンパイアサバイバー系の見下ろし型サバイバルアクション
- **構成**: 3ファイル（HTML5 canvas、ビルド・外部ライブラリ不要）
  - `index.html` … ページの骨組み（canvas・回転案内・エラー表示）
  - `style.css` … 見た目（ネオン風の光彩など）
  - `game.js` … ゲーム本体のロジック（約4,000行）。`type="module"` にはしない（`file://` でのダブルクリック起動を保つため）
- **勝利条件**: ミニボスを5体順に撃破するとラスボス「オーバーロード」が出現。撃破でクリア（時間経過での無条件出現は廃止 — ユーザー要望）
- **敗北条件**: HPが0になる（「死の免除」があれば1回だけ耐える）

## 3. ゲームの流れ

1. 雑魚敵がウェーブ制でだんだん強く・多くなる（敵の種類が入れ替わることで強くなる。HPは時間経過でスケールしない固定値、攻撃力・速度のみ時間経過でスケールし`CLEAR_TIME`＝540秒で頭打ち — ユーザー要望）
2. ウェーブの境目で**ミニボス**が5種ローテで順番に出現（生存中のミニボス／ラスボスがいる間は次を出さない — 同時出現禁止・ユーザー要望）
3. ミニボス撃破などで**宝箱**が出現 → 1個目で無敵ダッシュを獲得、2個目以降は毎回クールダウンが半分になる
4. ミニボスを**5体撃破**すると**ラスボス「オーバーロード」**出現（時間経過での無条件出現は廃止 — ユーザー要望）
5. 撃破で勝利画面（スコア表示）。クリア直後3秒は入力を無視（誤操作防止・ユーザー要望）

## 6. ダメージ計算式

武器ダメージは `weaponDmg(w)` ヘルパー（game.js 内）で一元計算：

```
最終ダメージ = w.dmg × dmgMul
```

その後 `damageEnemy` でさらに補正：

- **会心**: critChance の確率で **2.2倍**（上限50%）
- **バーサーク**: HP40%未満で +50% × 取得数（最大2）
- **コア露出**: ラスボスのコアフェーズ明けは被ダメ **1.6倍**（`coreExposed`）

例外: ストライカー（ダッシュ突撃）は `35 × strikerLv × dmgMul` の独自計算（武器ではないので上記の武器ダメージ式は通らない）。

## 7. レベルアップ（アップグレード）

XPを集めるとレベルアップし、3択から1つ選ぶ。武器系が最低1枚、未取得が1枚保証される。進化が出せる状態なら最優先で1枚入る。

**選択回数を減らしつつ1回あたりの伸びを強化**（ユーザー要望）: 必要XP `xpNext = round((5 + lv×3.8 + lv²×0.2) / 0.56)` でレベルアップ間隔を広げ、そのぶんステータス系パッシブの上昇量を全体的に増量した（下表）。式全体を当初 `/0.8` にすることでレベルの上がりやすさを0.8倍に抑えていたが、「もう少しレベル上がりづらくしたい」というユーザー要望を受けてさらに0.7倍（`0.8 × 0.7 = 0.56`）に調整し、`/0.56` とした。

**選択直後2秒無敵**（ユーザー要望）: 選択画面を閉じてプレイに戻った瞬間、`p.invuln` を2秒に設定。選択中に敵に囲まれていても不意打ちで即被弾しない。

## 8. 宝箱（1個目=無敵ダッシュ、2個目以降=CD半減）

4段ラダー方式は廃止し、よりシンプルな2パターンに再編（ユーザー要望）：

1. 🛡️ **無敵ダッシュ**（1個目）— Space/⚡ボタンで発動。無敵になって短距離を一気に駆け抜ける（初期CD6秒）
2. 🔋 **クールダウン短縮**（2個目以降・毎回）— 開けるたびにスキルCDが**そのつど半分**になる（6→3→1.5→0.75秒…と際限なく短縮）

開けるたびにHP15回復もおまけで付く。

## 9. 敵（10種）

**HPは時間経過でスケールしない**（下表のHPがそのまま最終値）。「だんだん強い敵が出てくる」演出はENEMY_SCHEDによる敵種の入れ替わりで表現する（旧仕様は時間経過の二次関数的HP倍率を持っていたが、撃破に手間取るほど敵がどんどん硬くなる「無限硬化（死のスパイラル）」の原因になっていたため撤去 — ユーザー要望）。攻撃力・速度は時間 t を `tc = min(t, 540)` にクランプした上でスケーリング: 攻撃力 `(1 + tc/200 + tc²/900000) × (1 + tc/540)`、速度 `(1 + tc/700) × (1 + (tc/540)×0.8)`（いずれも全体 **0.8倍** の緩和つき・ユーザー要望による調整）。**モブ敵（miniboss・ラスボスを除く9種）は上記の速度にさらに1.3倍**（ユーザー要望）。**`CLEAR_TIME`（540秒）で頭打ち**にする。XP だけは頭打ちなしで `基礎 × (1 + t/200)`（時間なりに伸ばして長期戦を救済。**ミニボスのみ専用の急カーブ** — 下記セクション10参照）。

**XPの獲得は自動**（ユーザー要望「経験値は自動で入るようにして、緑経験値玉なしにしよう」）: 敵を倒すと `killEnemy` 内で即座に `gainXP(e.xp)` が呼ばれ、XPがそのままプレイヤーに加算される。旧仕様にあった緑色のXPジェムをドロップ→吸引範囲に入ったら回収、という手順は完全に廃止した（初期化・ドロップ・吸引update・削除フィルタ・描画・AI自動プレイロジックなど関連コードを全箇所削除）。プレイヤーの `pickupRange`（175）は現在は**回復オーブの吸引専用**で、XPの取得可否には影響しない。

## 10. ミニボス（5種ローテ）

ウェーブ境界で出現。スキル入手ペースに合わせた順番（1匹目はダッシュなしでも避けられる）。

**同時出現の禁止**: 生存中のミニボス／ラスボスがいる間は次のミニボスもラスボスも出現しない（ユーザー要望）。5体を順に撃破し終えるとラスボスが出現する（下記セクション11参照）。

1. **弾幕型 spreader**（水色）— 包囲リング弾
2. **瞬移型 blinker**（黄）— テレポート
3. **爆撃型 bomber**（橙）— 範囲爆撃
4. **狙撃型 sniper**（緑・個別強化）— 一定距離を旋回しつつ、3-way扇状弾（角度±0.13rad）を1バーストにつき通常8発・猛攻モード10発連射する弾幕強化版。タメ0.35秒→バースト間隔0.20秒（猛攻0.16秒）で連射→CD1.8秒（猛攻1.3秒）。弾速450（猛攻500）・ダメージ20/発。「攻撃が当たる気がしない・手数が少ない」というユーザー要望を受け、単発の連射数増加ではなく扇状化で回避しにくさも同時に強化した
5. **突進型 charger**（赤・r55＝通常0.8倍サイズ・個別強化）— **1.5秒のタメ**を挟んでから、その瞬間のプレイヤー位置へ照準して長距離突進（速度480＝プレイヤー初期速度210の約2.3倍）。タメ中は歩いて距離を取るだけでは避けられず、ダッシュ等での回避が前提。ローテ最後のボスとしてタメ短縮・速度アップ・CD短縮（8.0→6.0秒）で高頻度化（ユーザー要望）。見た目はプレイヤーへ向いた矢印形で、丸いコアで覆わない

**突進の予兆演出（ユーザー要望「突進は1回光るなど合図の後してほしい」）**: タメ中（`chargeWarn`）は0.1秒周期で明滅させて「そろそろ来る」ことを伝え、タメが終わって実際に突進を開始する瞬間だけ強く1回光らせる（`hitFlash = 0.25`）＋バースト演出＋画面シェイク。ラスボスの突進技も同じ`chargeWarn`の仕組みを流用しており同様に光る。

**猛攻モード（HP50%フェーズ2）**: HPが最大値の50%を下から通過した瞬間に1回だけ発動。5種共通で攻撃が強化される（弾数・連射数の増加、CD短縮、予兆時間の短縮など個別調整）（ユーザー要望）。

**出現順スケーリング（ユーザー要望：終盤に出てくるやつほど少しずつ強くなる形にする。ラスボスHPだけを吊り上げる単発調整はしない）**: 基礎HP8000・基礎攻撃力30（弾幕系ダメージも含む）に対し、出現順インデックス（0〜4）に応じて2系統の倍率を掛ける。基礎HPは2000だったが、実プレイで「5秒かからず倒せてしまう・弱すぎる」というユーザー要望を受けて**8000（4倍）**へ引き上げ、目安30秒程度で倒せる強さにした。

- HP倍率 `orderMul = 1 + index × 0.28`（1匹目=1.00倍 → 5匹目=2.12倍。基礎HP4倍化と合わせて底上げ、旧0.19から引き上げ）
- 攻撃力倍率 `dmgOrderMul = 1 + index × 0.13`（1匹目=1.00倍 → 5匹目=1.52倍。接触ダメージに加え、enemyRing／爆弾／狙撃弾など弾幕系ダメージにも同率を適用）

プレイヤーレベル7以上で1レベルごとにHP+8%補正（`mbLvScale`）。**ただしレベル差20で頭打ち（最大+160%＝2.6倍）**（無限硬化の死のスパイラル防止 — 撃破が長引くほど雑魚狩りでレベルが上がり、次のミニボスがさらに硬くなる連鎖を断つため）。実際のHPは `8000 × mbLvScale × orderMul`。

**撃破XP**: `84 × (1 + t/120)`（専用式。後のボスほど報酬が増える）。**撃破と同時にXPを自動加算する**（旧仕様の「ジェム12個をばらまいて拾わせる」演出は廃止 — ユーザー要望「経験値は自動で入るようにして、緑経験値玉なしにしよう」。詳細は上記セクション9末尾参照）。

## 11. ラスボス「オーバーロード」

- 出現: **ミニボス5体を順に撃破し終えた後**（時間経過での無条件出現は廃止・ユーザー要望）。生存中のミニボスがいる間は出現しない
- HP: **ミニボスHPと連続性のある式**（ラスボスHPだけを孤立した数値として吊り上げる調整はしない・ユーザー要望）。ミニボス基礎HPの引き上げ（2000→8000）・出現順倍率の引き上げ（`orderMul`旧0.19→0.28）に連動させて同率で再計算している。固定基礎HP `mbBaseHp = 7600` に「6体目」相当の出現順倍率 `finalMbOrderMul = 1 + 4 × 0.28 = 2.12` と係数20を掛けた `7600 × 2.12 × 20 ≈ 322,240` をベースに（プレイヤーLv9以上で1レベルごと+9%。**ただしレベル差20で頭打ち＝最大+180%＝2.8倍**、無限硬化の死のスパイラル防止）。実プレイで「ラスボスも短時間で倒せてしまう」というユーザー要望を受けて、目安1分程度で倒せる強さまで引き上げた（過去に「以後変更しない」としていたが、今回改めてユーザーから調整要望があったため変更）
- 攻撃: 弾幕・突進・爆撃・瞬移の融合技
- **コアフェーズ**: HPが **65% / 35%** を切るたびに1回ずつ発動。無敵化してアリーナを侵食（約6.5秒、猶予・演出強化済み）。耐え抜くと **4秒間コア露出**（被ダメ1.6倍）＝反撃チャンス
- 撃破で勝利

## 14. 設計方針

- **ビルド不要主義**: `index.html` をブラウザで開けばすぐ遊べる状態を保つ（バンドラ・モジュール・外部CDNは使わない。ファイルは HTML/CSS/JS の3分割まで）
- **バックアップファイルは作らない**: 履歴は git に任せる（`index.html.bak` などは作らない）
- **爽快感優先のバランス**: 進化・宝箱ラダーなどで「強くなった実感」を出す。敵の攻撃力・速度は0.8倍緩和済み（遊びやすさ優先）
- **難所は段階的に**: ミニボスの順番はスキル入手ペースに合わせる（詰み防止）
- **スマホ発熱への配慮**: タッチ端末（`IS_TOUCH`）では描画解像度（DPR上限1.5）・パーティクル数（半減）・描画フレームレート（60fps上限）・影（`shadowBlur`を常に0化）を抑え、高リフレッシュレート機での過剰な描画負荷を避ける（ユーザー要望。PC版はネオン風の影表現をそのまま維持）
- **変更したら必ず実機検証**: `.claude/skills/verify/SKILL.md` の手順で実際にプレイして確認する
