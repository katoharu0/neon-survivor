# Neon Survivor 仕様書・方針書

このファイルは **ゲームの現在の仕様（数値・仕組み）と設計方針** をまとめたものです。

> **📌 更新ルール（最重要）**

> コードと仕様書がズレたら、仕様書は信頼できなくなる。「コードを変えたら SPEC.md も直す」をワンセットにする。

## 1. 概要

- **ジャンル**: ヴァンパイアサバイバー系の見下ろし型サバイバルアクション
- **構成**: 3ファイル（HTML5 canvas、ビルド・外部ライブラリ不要）
  - `index.html` … ページの骨組み（canvas・回転案内・エラー表示）
  - `style.css` … 見た目（ネオン風の光彩など）
  - `game.js` … ゲーム本体のロジック（約4,000行）。`type="module"` にはしない（`file://` でのダブルクリック起動を保つため）
- **勝利条件**: 9分（540秒）生存するとラスボスが出現。撃破でクリア（実質約10分）
- **敗北条件**: HPが0になる（「死の免除」があれば1回だけ耐える）

## 2. 操作

| 操作 | キー / タッチ |
|---|---|
| 移動 | WASD / 矢印キー / 画面タッチ（バーチャルスティック、指を保持したまま方向変更可） |
| スキル（ダッシュ等） | Space / 画面の⚡ボタン（宝箱で入手後） |
| ポーズ | Esc / P |

## 3. ゲームの流れ

1. 雑魚敵がウェーブ制でだんだん強く・多くなる（時間経過でHP・攻撃力・速度がスケール、クリア時点でちょうど×2の補正）
2. ウェーブの境目で**ミニボス**が出現（5種ローテ）
3. ミニボス撃破などで**宝箱**が出現 → スキルが5段階で強化される
4. 540秒（`CLEAR_TIME`）で**ラスボス「オーバーロード」**出現
5. 撃破で勝利画面（スコア表示）

| オービット orbit | 🛰 | 2個 / ダメージ10 / 半径80 / 回転2.6 | サテライトリング（半径1.5倍・回転1.7倍・+1個） |

## 6. ダメージ計算式

武器ダメージは `weaponDmg(w)` ヘルパー（game.js 内）で一元計算：

```
最終ダメージ = w.dmg × dmgMul × (オーバードライブ中なら 1.3)
```

その後 `damageEnemy` でさらに補正：

- **会心**: critChance の確率で **2.2倍**（上限50%）
- **バーサーク**: HP40%未満で +50% × 取得数（最大2）
- **コア露出**: ラスボスのコアフェーズ明けは被ダメ **1.6倍**（`coreExposed`）

例外: ストライカー（ダッシュ突撃）は `35 × strikerLv × dmgMul` の独自計算（武器ではないのでオーバードライブ倍率は乗らない）。

## 7. レベルアップ（アップグレード）

XPを集めるとレベルアップし、3択から1つ選ぶ。武器系が最低1枚、未取得が1枚保証される。進化が出せる状態なら最優先で1枚入る。

ステータス系パッシブ：

| 名前 | 効果 | 上限 |
|---|---|---|
| 体力増強 | 最大HP+30（+30回復） | なし |
| 俊足 | 移動速度+10% | なし |
| 攻撃力上昇 | 全武器ダメージ+12%（dmgMul） | なし |
| 再生 | 毎秒HP+0.9 | regen 6.0 |
| 会心 | 会心率+8%（2.2倍ダメージ） | 50% |
| 装甲 | 被ダメ-9% | 45% |
| 強欲の輝き | 30秒XP+60%（重ねがけ延長、最大60秒） | — |
| 高速詠唱 | 全武器の発動8%高速化 | — |
| スパイク装甲 | 接触反射ダメージ+16 | — |
| 回避 | 被弾10%で完全回避 | 30% |
| バリア | 20秒ごとに被弾1回無効 | 1回取得 |
| バーサーク | HP40%未満で全ダメージ+50% | 2回 |
| アドレナリン | 低HPほど移動速度+（HP0で+15%×段階） | 3回 |
| 死の免除 | 1回だけHP1で生き残る | 1回 |

| 起爆 | 撃破時に確率で爆発連鎖（10%×段階） | 2回 |

## 8. 宝箱（スキル強化ラダー・全5段）

宝箱は開けるたびに**順番どおり**に1段ずつ強化＋HP15回復。6個目以降は「フルリペア（全回復）」。

1. ⚡ **フェイズダッシュ** — Space/⚡で短距離加速（CD6秒）
2. 🛡️ **ゴーストダッシュ** — ダッシュ中無敵
3. 🔋 **スキルチャージャー** — CD半分（6→3秒）
4. 🌀 **ブリンク** — ダッシュが瞬間移動に進化（無敵つき）
5. 💠 **オーバーチャージ** — CDさらに半分（3→1.5秒）

## 9. 敵（11種）

時間 t によるスケーリング: HP `(1 + t/52 + t²/500000) × (1 + t/540)`、攻撃力・速度は全体 **0.8倍** の緩和つき（ユーザー要望による調整）。XP は `基礎 × (1 + t/200)`（**ミニボスのみ専用の急カーブ** — 下記セクション10参照）。

| 敵 | HP基礎 | 速度基礎 | 攻撃 | 特徴 |
|---|---|---|---|---|
| grunt | 24 | 68 | 14 | 基本の雑魚 |
| swift | 16 | 142 | 11 | 速い |
| tank | 120 | 58 | 25 | 硬い |
| splitter | 46 | 60 | 15 | 倒すと分裂 |
| spitter | 38 | 54 | 15 | 涙滴弾を撃つ |
| weaver | 22 | 110 | 11 | 蛇行移動 |
| dasher | 42 | 58 | 20 | 突進 |
| bomber | 37 | 74 | 31 | 自爆（導火つき） |
| orbiter | 30 | 120 | 14 | 距離を保ち3-way拡散弾 |
| brute | 190 | 52 | 42 | 鈍重・叩きつけ攻撃 |
| miniboss | 520 | 76 | 30 | 下記ローテ参照 |

**エリート**: 確率で出現。HP3.2倍・サイズ1.5倍・攻撃1.4倍・XP4倍。

## 10. ミニボス（5種ローテ）

ウェーブ境界で出現。スキル入手ペースに合わせた順番（1匹目はダッシュなしでも避けられる）：

1. **弾幕型 spreader**（水色）— 包囲リング弾
2. **突進型 charger**（赤・大型r69・鈍足0.85倍）— ダッシュ必須の突進
3. **爆撃型 bomber**（橙）— 範囲爆撃
4. **瞬移型 blinker**（黄）— テレポート
5. **狙撃型 sniper**（緑）— 一定距離を旋回しつつ狙撃

プレイヤーレベル7以上で1レベルごとにHP+8%補正。

**撃破XP**: `84 × (1 + t/120)`（専用式。従来 `28 × (1 + t/200)` の **3.6〜4.4倍**、後のボスほど報酬が増える — ユーザー要望）。出現時刻ごとの実値: 147 / 210 / 273 / 336 / 399。ジェムは **12個** に分散（合計XPは式のとおり。コンボ継続中は最大2倍）。

## 11. ラスボス「オーバーロード」

- 出現: `CLEAR_TIME`（540秒）経過時
- HP: **16900** ×（プレイヤーLv9以上で1レベルごと+9%）
- 攻撃: 弾幕・突進・爆撃・瞬移の融合技
- **コアフェーズ**: HPが **65% / 35%** を切るたびに1回ずつ発動。無敵化してアリーナを侵食（約6.5秒、猶予・演出強化済み）。耐え抜くと **4秒間コア露出**（被ダメ1.6倍）＝反撃チャンス
- 撃破で勝利

## 12. オーバードライブ

- 雑魚を**6秒以内の間隔で15体連続キル**すると発動
- 効果: **3.0秒間** 武器ダメージ **1.3倍**（`OVERDRIVE_DMG_MUL`）＋移動速度 **1.15倍**（`OVERDRIVE_SPD_MUL`）
- クールダウン: 14秒

## 15. 設計方針

- **ビルド不要主義**: `index.html` をブラウザで開けばすぐ遊べる状態を保つ（バンドラ・モジュール・外部CDNは使わない。ファイルは HTML/CSS/JS の3分割まで）
- **バックアップファイルは作らない**: 履歴は git に任せる（`index.html.bak` などは作らない）
- **爽快感優先のバランス**: コンボ・オーバードライブ・進化などで「強くなった実感」を出す。敵の攻撃力・速度は0.8倍緩和済み（遊びやすさ優先）
- **難所は段階的に**: ミニボスの順番はスキル入手ペースに合わせる（詰み防止）
- **変更したら必ず実機検証**: `.claude/skills/verify/SKILL.md` の手順で実際にプレイして確認する
- **変更したらこの SPEC.md を同じコミットで更新する**
